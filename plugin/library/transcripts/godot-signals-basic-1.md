**Conflict named** — Point 6's "parent holds an `@onready` ref typed to the child's concrete `class_name`" partially conflicts with **godot-composition rule 5** (dependencies injected via `@export`, never reach concrete types) and **rule 6** (depend on signals/duck-typing, not `is Player`). Use the `class_name`/typing benefit; inject the dependency, don't hardcode the concrete child type. No event-bus autoload appears in pt.1, so no rule-5-autoload conflict here (that was pt.2's point 2).
