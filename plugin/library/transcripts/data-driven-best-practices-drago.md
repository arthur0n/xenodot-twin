## What the video teaches

Author enemy STATS as data, not hard-coded per-script. Shows BOTH paths then declares the `.tres` path "the real Godot way": one base script reads the data and pushes stats down to ability child nodes; new enemy = new data file, no code. Storage choice (JSON vs `.tres`) is the video's spine.
