## What the video actually is

A **motivation / methodology** talk, NOT a data-driven implementation tutorial. Thesis: build isolated, reusable SYSTEMS (controller, ability-unlock, game-state, combat, level-gating) instead of throwaway small games; design each for EXPANSION from day one. The only concrete architecture claim: rebuild the player controller as a **modular state machine using composition** so new abilities don't force rewrites. No Resource/`.tres`/data-asset technique is shown. "Data-driven" in our brief is the lens; the video itself never reaches it.
